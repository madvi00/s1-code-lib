GETTING STARTED

시작하기

이 라이브러리를 프로젝트에 처음 적용할 때 필요한 단계별 가이드입니다. assets 폴더는 스타일(CSS)과 동작(JS)만 제공하므로, 각 컴포넌트 페이지의 코드 스니펫에서 HTML 마크업도 함께 복사해서 사용해야 합니다.

1
assets 폴더 다운로드
아래 버튼으로 assets 폴더를 다운로드한 후 프로젝트 루트에 압축을 풉니다. CSS, JS, 이미지 등 필요한 파일이 모두 포함되어 있습니다.
assets.zip 다운로드
project/ ├── assets/ │ ├── css/ │ │ ├── lib-style.css ← 공통 필수 │ │ ├── pc-style.css ← PC 전용 │ │ ├── mobile-style.css ← Mobile 전용 │ │ └── lib/ │ ├── js/ │ │ ├── lib/ │ │ │ ├── jquery-3.6.0.min.js │ │ │ └── swiper-bundle.min.js │ │ ├── dropdown.js │ │ ├── modal.js │ │ └── ... │ └── images/ └── index.html
2
CSS 연결
PC와 Mobile은 사용하는 CSS 파일이 다릅니다. lib-style.css는 공통 필수이며, 플랫폼에 맞는 컴포넌트 스타일 파일을 함께 연결합니다.
<!-- 공통 기본 스타일 (필수) --> <link rel="stylesheet" href="assets/css/lib-style.css" /> <!-- PC 컴포넌트 스타일 --> <link rel="stylesheet" href="assets/css/pc-style.css" /> <!-- Datepicker 사용 시 추가 --> <link rel="stylesheet" href="assets/css/lib/bootstrap-datepicker.standalone.css" />
<!-- 공통 기본 스타일 (필수) --> <link rel="stylesheet" href="assets/css/lib-style.css" /> <!-- Mobile 컴포넌트 스타일 --> <link rel="stylesheet" href="assets/css/mobile-style.css" /> <!-- Datepicker 사용 시 추가 --> <link rel="stylesheet" href="assets/css/lib/bootstrap-datepicker.standalone.css" />
3
JS 연결
jQuery는 공통 필수입니다. 인터랙션이 있는 컴포넌트는 해당 JS 파일을 추가로 연결해야 동작합니다. 사용하지 않는 컴포넌트의 JS는 포함하지 않아도 됩니다.
<!-- jQuery (필수) --> <script src="assets/js/lib/jquery-3.6.0.min.js"></script> <!-- 컴포넌트별 JS (사용하는 것만 추가) --> <script src="assets/js/dropdown.js"></script> <!-- Dropdown, Filter Chip --> <script src="assets/js/modal.js"></script> <!-- Modal --> <script src="assets/js/datepicker.js"></script> <!-- Datepicker --> <script src="assets/js/time-picker.js"></script> <!-- Time Picker --> <script src="assets/js/range-time-picker.js"></script> <!-- Range Time Picker --> <script src="assets/js/tab.js"></script> <!-- Tab --> <script src="assets/js/route-accordion.js"></script> <!-- Route Accordion -->
<!-- jQuery (필수) --> <script src="assets/js/lib/jquery-3.6.0.min.js"></script> <!-- Swiper (TimePicker 등 사용 시) --> <script src="assets/js/lib/swiper-bundle.min.js"></script> <!-- 컴포넌트별 JS (사용하는 것만 추가) --> <script src="assets/js/dropdown.js"></script> <!-- Dropdown, Filter Chip --> <script src="assets/js/multi-dropdown.js"></script> <!-- Multi Dropdown --> <script src="assets/js/modal.js"></script> <!-- Modal --> <script src="assets/js/datepicker.js"></script> <!-- Datepicker --> <script src="assets/js/time-picker.js"></script> <!-- Time Picker --> <script src="assets/js/range-time-picker.js"></script> <!-- Range Time Picker --> <script src="assets/js/tab.js"></script> <!-- Tab --> <script src="assets/js/route-accordion.js"></script> <!-- Route Accordion --> <script src="assets/js/input.js"></script> <!-- Input -->
💡 Badge, Button, Checkbox, Radio, Switch, Loading, Weather Widget은 JS 없이 CSS만으로 동작합니다.
4
컴포넌트 HTML 복사
원하는 컴포넌트 페이지로 이동한 후 코드 스니펫의 HTML 마크업을 복사해 페이지에 붙여넣습니다. CSS·JS만으로는 화면에 아무것도 표시되지 않으며, HTML 마크업이 반드시 필요합니다.
<!-- 예시: Button 컴포넌트 --> <button type="button" class="btn btn-blue"> <span>라벨</span> </button>

DESIGN TOKENS

디자인 토큰

에스원 디자인 시스템은 Foundation → Semantic 2계층 구조로 토큰을 관리합니다. Foundation은 원시 값이며, Semantic은 사용 맥락에 맞게 Foundation 토큰을 참조합니다. 컴포넌트에는 반드시 Semantic 토큰을 사용하세요.

Layer 1
Foundation Token
색상, 폰트, 간격 등의 원시(raw) 값을 정의합니다. 직접 사용하지 않습니다.
--foundation-color-blue-400: #1d6ceb
Layer 2
Semantic Token
사용 맥락(버튼, 폼, 텍스트 등)에 따라 Foundation 토큰을 참조합니다.
--semantic-color-button-background-primary-default: var(--foundation-color-blue-400)
적용
Component
컴포넌트 CSS에 Semantic 토큰을 적용합니다.
background: var(--semantic-color-button-background-primary-default)

사용 가이드라인

컴포넌트의 올바른 사용법, 색상·간격 원칙, 접근성(Accessibility) 지침을 제공합니다.
일관된 인터페이스 구축을 위해 아래 기준을 따라 주세요.

Button — Do's & Don'ts

✅ Do
페이지 내 가장 중요한 액션 하나에만 Primary 버튼을 사용하고, 보조 액션에는 Secondary를 사용하세요.
❌ Don't
Primary 버튼을 여러 개 나란히 배치하지 마세요. 사용자가 주요 액션을 파악하기 어려워집니다.
✅ Do
버튼 레이블은 동사로 시작하는 명확한 행동 언어를 사용하세요. "저장", "검색", "초기화" 등 결과를 예측할 수 있어야 합니다.
❌ Don't
의미 없거나 모호한 레이블 사용을 피하세요. 사용자가 버튼의 결과를 예측하기 어렵습니다.
✅ Do
되돌릴 수 없는 위험한 액션(삭제, 초기화)에는 Danger 버튼을 사용하여 사용자에게 경고를 전달하세요.
❌ Don't
삭제와 같은 위험 액션에 Primary 버튼을 사용하지 마세요. 사용자가 행동의 심각성을 인지하기 어렵습니다.

❌ Don't

접근성 (Accessibility) 가이드

컴포넌트 지침 코드 예시
Button 버튼의 목적이 텍스트만으로 불분명할 경우 aria-label을 추가하세요. 아이콘 전용 버튼에는 반드시 필요합니다. <button aria-label="팝업창 닫기">
Input 모든 입력 필드에는 연결된 <label>이 필요합니다. 시각적으로 숨겨야 할 경우 sr-only 등 숨김 클래스를 사용하세요. <label for="id" class="sr-only">
Checkbox / Radio for와 id 속성을 항상 일치시켜 클릭 영역을 label 전체로 확장하세요. <label for="chk1"><input id="chk1">
Dropdown 트리거 버튼에 aria-haspopup="listbox"와 aria-expanded를 적용하고, 목록 열림·닫힘 상태를 동기화하세요. aria-haspopup="listbox" aria-expanded="true"
Modal 모달이 열릴 때 포커스를 내부로 이동하고, 닫힐 때 트리거 버튼으로 반환하세요. ESC 키 닫기를 지원해야 합니다. role="dialog" aria-modal="true"
Switch 현재 상태를 스크린 리더가 읽을 수 있도록 role="switch"와 aria-checked를 사용하세요. role="switch" aria-checked="true"
Tab 탭 컨테이너에 role="tablist", 각 탭에 role="tab", 탭 패널에 role="tabpanel"을 적용하세요. role="tab" aria-selected="true"
Loading 로딩 스피너에 role="status"와 aria-live="polite"를 사용하여 스크린 리더에 상태 변화를 알리세요. role="status" aria-live="polite"
Datepicker 날짜 입력 필드에 aria-label로 날짜 형식을 안내하고, 달력 팝업에 role="dialog"를 적용하세요. aria-label="날짜 선택 (YYYY.MM.DD)"
공통 키보드 포커스 스타일을 제거하지 마세요. :focus-visible 스타일이 명확히 보여야 하며, 텍스트와 배경 간 명도 대비율 4.5:1 이상을 유지하세요. :focus-visible { outline: 2px solid #1d6ceb; }

HTML 코드 규칙

들여쓰기 · 빈 줄

  • 들여쓰기는 공백 2칸을 사용한다. 탭(Tab) 문자는 사용하지 않는다.
  • 가독성을 위한 빈 줄은 허용하되, 과도한 빈 줄 사용은 지양한다.

주석

  • 영역 주석은 시작과 끝을 모두 작성하며, 닫는 주석은 // 를 앞에 붙인다.
  • 수정 주석은 HTML에만 작성하며, YYYY-MM-DD 내용 형식을 따른다.
  • 이미 개발에 전달된 후 재수정 시에도 새 날짜로 별도 주석을 추가한다.
  • 마크업이 제거될 경우 닫는 주석만 남기고 내용을 삭제한다. 재사용 가능성이 있으면 주석 처리만 한다.

  • JavaScript 조작용 클래스는 js- 접두사를 붙이고, 스타일 적용 목적으로는 사용하지 않는다.

작성 규칙

  • 선택자와 여는 중괄호 사이에 공백 1칸을 둔다. 예) .class-name {
  • 콜론(:)과 속성값 사이에 공백 1칸을 둔다. 예) margin-left: 0;
  • 모든 선언은 세미콜론(;)으로 마무리한다. (마지막 속성 포함)
  • 닫는 중괄호는 단독 줄에 위치한다.
  • 코드 그룹 간 빈 줄은 1줄만 허용한다. 종료 주석은 작성하지 않는다.
  • * 전체 선택자는 남용하지 않는다. 리셋 목적의 단순 사용은 허용한다.
  • 인라인 style 속성은 동적 제어가 필요한 경우에만 제한적으로 허용한다.
  • 단위는 px, rem, %를 상황에 맞게 사용하되 border에는 px를 사용한다.

접근성 작업 시 유의사항

  • 키보드 초점(focus)이 논리적인 순서로 이동하도록 마크업 순서와 시각적 순서를 일치시킨다.
  • 모든 이미지에 alt 속성을 제공한다. 장식용 이미지는 alt=""로 처리하고, 내용이 있는 이미지는 의미 있는 설명을 작성한다.
  • 색상만으로 정보를 전달하지 않는다. 텍스트, 아이콘, 패턴 등 보조 수단을 함께 사용한다.
  • 버튼에는 반드시 type 속성을 명시한다. 기본값은 submit이므로 일반 버튼은 type="button"을 명시해야 한다.
  • 링크(<a>)는 목적지나 기능을 명확히 전달하는 텍스트를 포함해야 한다. "여기를 클릭", "자세히" 같은 모호한 텍스트는 사용하지 않는다.
  • 팝업(모달)이 열리면 초점을 팝업 내부로 이동시키고, 닫히면 팝업을 열었던 요소로 초점을 복귀시킨다.
  • 스크린 리더 전용 텍스트는 .sr-only 클래스를 사용하여 화면에는 보이지 않지만 보조 기술에는 노출되도록 처리한다.
  • 콘트라스트 비율은 텍스트 기준 4.5:1 이상(대형 텍스트 3:1 이상)을 준수한다.
/* 스크린 리더 전용 숨김 텍스트 */
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  margin: -1px;
  padding: 0;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  clip-path: inset(50%);
  white-space: nowrap;
  border: 0;
}
