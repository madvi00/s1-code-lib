GETTING STARTED

시작하기

이 라이브러리를 프로젝트에 처음 적용할 때 필요한 단계별 가이드입니다. assets 폴더는 스타일(CSS)과 동작(JS)만 제공하므로, 각 컴포넌트 페이지의 코드 스니펫에서 HTML 마크업도 함께 복사해서 사용해야 합니다.

1
assets 폴더 다운로드
아래 버튼으로 assets 폴더를 다운로드한 후 프로젝트 루트에 압축을 풉니다. CSS, JS, 이미지 등 필요한 파일이 모두 포함되어 있습니다.
assets.zip 다운로드
project/ ├── assets/ │ ├── css/ │ │ ├── lib-style.css ← 공통 필수 │ │ ├── pc-style.css ← PC 전용 │ │ ├── mobile-style.css ← Mobile 전용 │ │ └── lib/ │ ├── js/ │ │ ├── lib/ │ │ │ ├── jquery-3.6.0.min.js │ │ │ └── swiper-bundle.min.js │ │ ├── dropdown.js │ │ ├── modal.js │ │ └── ... │ └── images/ └── index.html
2
CSS 연결
PC와 Mobile은 사용하는 CSS 파일이 다릅니다. lib-style.css는 공통 필수이며, 플랫폼에 맞는 컴포넌트 스타일 파일을 함께 연결합니다.
<!-- 공통 기본 스타일 (필수) --> <link rel="stylesheet" href="assets/css/lib-style.css" /> <!-- PC 컴포넌트 스타일 --> <link rel="stylesheet" href="assets/css/pc-style.css" /> <!-- Datepicker 사용 시 추가 --> <link rel="stylesheet" href="assets/css/lib/bootstrap-datepicker.standalone.css" />
<!-- 공통 기본 스타일 (필수) --> <link rel="stylesheet" href="assets/css/lib-style.css" /> <!-- Mobile 컴포넌트 스타일 --> <link rel="stylesheet" href="assets/css/mobile-style.css" /> <!-- Datepicker 사용 시 추가 --> <link rel="stylesheet" href="assets/css/lib/bootstrap-datepicker.standalone.css" />
3
JS 연결
jQuery는 공통 필수입니다. 인터랙션이 있는 컴포넌트는 해당 JS 파일을 추가로 연결해야 동작합니다. 사용하지 않는 컴포넌트의 JS는 포함하지 않아도 됩니다.
<!-- jQuery (필수) --> <script src="assets/js/lib/jquery-3.6.0.min.js"></script> <!-- 컴포넌트별 JS (사용하는 것만 추가) --> <script src="assets/js/dropdown.js"></script> <!-- Dropdown, Filter Chip --> <script src="assets/js/modal.js"></script> <!-- Modal --> <script src="assets/js/datepicker.js"></script> <!-- Datepicker --> <script src="assets/js/timePicker.js"></script> <!-- Time Picker --> <script src="assets/js/rangeTimePicker.js"></script> <!-- Range Time Picker --> <script src="assets/js/tab.js"></script> <!-- Tab --> <script src="assets/js/routeAccordion.js"></script> <!-- Route Accordion -->
<!-- jQuery (필수) --> <script src="assets/js/lib/jquery-3.6.0.min.js"></script> <!-- Swiper (TimePicker 등 사용 시) --> <script src="assets/js/lib/swiper-bundle.min.js"></script> <!-- 컴포넌트별 JS (사용하는 것만 추가) --> <script src="assets/js/dropdown.js"></script> <!-- Dropdown, Filter Chip --> <script src="assets/js/multiDropdown.js"></script> <!-- Multi Dropdown --> <script src="assets/js/modal.js"></script> <!-- Modal --> <script src="assets/js/datepicker.js"></script> <!-- Datepicker --> <script src="assets/js/timePicker.js"></script> <!-- Time Picker --> <script src="assets/js/rangeTimePicker.js"></script> <!-- Range Time Picker --> <script src="assets/js/tab.js"></script> <!-- Tab --> <script src="assets/js/routeAccordion.js"></script> <!-- Route Accordion --> <script src="assets/js/input.js"></script> <!-- Input -->
💡 Badge, Button, Checkbox, Radio, Switch, Loading, Weather Widget은 JS 없이 CSS만으로 동작합니다.
4
컴포넌트 HTML 복사
원하는 컴포넌트 페이지로 이동한 후 코드 스니펫의 HTML 마크업을 복사해 페이지에 붙여넣습니다. CSS·JS만으로는 화면에 아무것도 표시되지 않으며, HTML 마크업이 반드시 필요합니다.
<!-- 예시: Button 컴포넌트 --> <button type="button" class="btn btn-blue"> <span>라벨</span> </button>

DESIGN TOKENS

디자인 토큰

에스원 디자인 시스템은 Foundation → Semantic 2계층 구조로 토큰을 관리합니다. Foundation은 원시 값이며, Semantic은 사용 맥락에 맞게 Foundation 토큰을 참조합니다. 컴포넌트에는 반드시 Semantic 토큰을 사용하세요.

Layer 1
Foundation Token
색상, 폰트, 간격 등의 원시(raw) 값을 정의합니다. 직접 사용하지 않습니다.
--foundation-color-blue-400: #1d6ceb
Layer 2
Semantic Token
사용 맥락(버튼, 폼, 텍스트 등)에 따라 Foundation 토큰을 참조합니다.
--semantic-color-button-background-primary-default: var(--foundation-color-blue-400)
적용
Component
컴포넌트 CSS에 Semantic 토큰을 적용합니다.
background: var(--semantic-color-button-background-primary-default)

GUIDELINES

사용 가이드라인

컴포넌트의 올바른 사용법과 접근성(Accessibility) 지침을 제공합니다.

Button — Do's & Don'ts

✅ Do
페이지 내 가장 중요한 액션 하나에만 Primary 버튼을 사용하고, 보조 액션에는 Secondary를 사용하세요.
❌ Don't
Primary 버튼을 여러 개 나란히 배치하지 마세요. 사용자가 주요 액션을 파악하기 어려워집니다.
✅ Do
버튼 레이블은 동사로 시작하는 명확한 행동 언어를 사용하세요. "확인", "저장", "검색" 등.
❌ Don't
의미 없거나 모호한 레이블 사용을 피하세요. 사용자가 버튼의 결과를 예측하기 어렵습니다.

접근성 (Accessibility) 가이드

컴포넌트 지침 코드 예시
Button 버튼의 목적이 텍스트만으로 불분명할 경우 aria-label을 추가하세요. 아이콘 전용 버튼에는 반드시 필요합니다. <button aria-label="팝업창 닫기">
Input 모든 입력 필드에는 연결된 <label>이 필요합니다. 시각적으로 숨겨야 할 경우 ir 클래스를 사용하세요. <label for="id" class="ir">
Checkbox/Radio 항상 for와 id 속성을 일치시켜 클릭 영역을 label 전체로 확장하세요. <label for="chk1"><input id="chk1">
Modal 모달 열릴 때 포커스를 모달 내부로 이동하고, 닫힐 때 트리거 버튼으로 포커스를 반환하세요. ESC 키 닫기 지원 필요. role="dialog" aria-modal="true"
Switch switch의 현재 상태를 스크린 리더가 읽을 수 있도록 role="switch"와 aria-checked를 사용하세요. role="switch" aria-checked="true"
Tab 탭 컨테이너에 role="tablist", 각 탭에 role="tab", 탭 패널에 role="tabpanel"을 적용하세요. role="tab" aria-selected="true"
공통 키보드 포커스 스타일을 제거하지 마세요. :focus-visible 스타일이 명확히 보여야 합니다. 최소 명도 대비율 4.5:1을 유지하세요. :focus-visible { outline: 2px solid blue; }
